    搭建博客的收获与建议

    约 761 个字 4 行代码 预计阅读时间 3 分钟

    搭建博客是一个具有一定挑战（emm...对于CS萌新来说，大佬随意~）但也非常有收获的过程。在这个过程中，能够学到许多新的知识，积累相关开发的经验，毕竟CS的学习始终是需要上手的。以下是我在搭建博客时的一些收获和建议：

    收获

    技术提升：

    提高了对前端语言（如HTML、CSS、JavaScript）的理解，虽然不会真的直接大量地去写，不过看得懂源码以及能够写一点JS，确实能让网页变得更漂亮智能。

    熟悉Markdown语法和Latex数学公式，并在撰写博文时熟练运用，Latex的数学公式是真的非常全，所以上手也有一定难度，遇到不清楚的就多查，用多了就记住了。

    内容创作：

    学习如何更好地组织和展示内容，巩固和呈现自己学习的知识。需要选取一些适合展示的内容，一般来说，写的过分详尽反而有点浪费时间而且也不太适合阅读，估计你自己都不太看得下去太大篇的文段。

    建议

    选择合适的平台：

    初学者可以选择一些简单易用的博客框架，比如mkdocs和hexo，以便快速上手。mkdocs纯文本如果你觉得太难看了可以选择hexo，不过如果你是单纯写总结mkdocs个人觉得更合适，至少git要会，建议部署在githubpage，服务器租也是要钱的，而且国内还需要备案。

    Mkdocs 简介

    1. 概述

    由于GitHub和Markdown的普及，很多时候我们可以用markdown来简便地写出文档，并且通过Mkdocs 和 GitHub Pages等工具快速地进行文档的部署。

    2. 安装mkdocs

    需要已下载Python和pip 

    pip install mkdocs
    各种插件也均是通过pip下载

    3. 配置mkdocs

    推荐 material 主题 配置yaml文件，不太需要完全懂yaml语法，只是要知道yaml和Python一样是靠缩进来区分代码块 详见 MkDocs 文档 里面什么都有

    4. 部署site

    选择一个文件夹 

    mkdocs new .
    在当前目录初始化一个新的 MkDocs 项目，里面结构为 
    - mkdocs.yml
- index.md
- docs/
    在docs文件夹里创建markdown文件就可以开始创作了

    mkdocs build
    生成site/ 接下来只需要将该文件夹里的内容上传即可。注意是上传文件夹里的内容，而不是直接上传文件夹。如果要绑定自己的域名，需要添加CNAME文件(类型是任意文本，先创建.txt再把后缀名删除)

    Note

    DNF 解析更新需要一定时间，运气不好可能需要等待近10min，请耐心等待。

    mkdocs serve
    本地预览网站效果，建议配置yaml的时候就跑起来，方便检查是否有语法错误以及插件是否正常生效，若有语法错误网站是解析不出来的。
