    JavaScript 原型链污染漏洞

    约 601 个字 13 行代码 预计阅读时间 2 分钟

    引言

    JavaScript 原型链（Prototype Chain）是实现对象继承的一种重要机制。然而，这种机制也可能带来一些安全隐患，其中最著名的就是原型链污染（Prototype Pollution）漏洞。

    概述

    在 JavaScript 中，每个对象都有一个内部链接到另一个对象，这个对象被称为其原型（prototype）。当我们访问对象的属性时，如果该属性不存在于对象自身，则 JavaScript 引擎会沿着原型链向上查找，直到找到该属性或到达原型链的顶端 'Object.prototype'。 

    let obj = {};
console.log(obj.toString); // [Function: toString]
    虽然 obj 对象本身没有 toString 方法，但它的原型链上有。

    原理

    原型链污染漏洞的核心在于攻击者能够通过不安全的输入操作，修改对象的原型，从而影响到所有通过该原型创建的对象。这通常发生在深度合并（deep merge）或对象克隆等操作中。

    举个例子： 

    let maliciousPayload = JSON.parse('{"__proto__": {"polluted": "yes"}}');
let target = {};
Object.assign(target, maliciousPayload);

console.log({}.polluted); // yes
    在这个示例中，maliciousPayload 对象包含一个特殊的属性 proto，用于修改目标对象 target 的原型链。通过 Object.assign 操作，target 对象的原型被污染，导致所有新创建的对象都受影响。

    影响

    1. 数据完整性：原型链污染可能导致对象属性意外改变，破坏应用的正常功能。
    2. 安全风险：攻击者可以利用原型链污染注入恶意代码，进一步执行跨站脚本攻击（XSS）或远程代码执行（RCE）。
    3. 系统稳定性：原型链污染可能引发难以调试的错误，影响系统的稳定性和可维护性。

    防护措施

    1. 避免直接使用不可信数据：在处理用户输入的数据时，应避免直接将其用于修改对象的原型。例如，深度合并操作时，应过滤掉 proto 属性。
    2. 使用安全的对象合并库：选择经过审查和测试的第三方库进行对象合并操作，例如 lodash 的 merge 函数，它已经处理了原型链污染问题。
    3. 启用严格模式：在严格模式下，某些情况下对原型链的修改会被禁止，可以一定程度上防止原型链污染。

    "use strict";
let maliciousPayload = JSON.parse('{"__proto__": {"polluted": "yes"}}');
let target = {};
Object.assign(target, maliciousPayload);

console.log({}.polluted); // undefined
    4. 定期审计代码：对代码进行安全审计，及时发现并修复潜在的安全漏洞。
