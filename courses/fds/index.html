    Stack

    • LIFO(Last-in-First-out)，意味着进出栈会倒序
    • 操作：
    • Push 入栈，将一个元素添加到栈顶
    • Pop 出栈，将栈顶元素弹出

    Queue

    • FIFO(First-in-First-out)
    • 操作：
    • Enqueue 入队，将一个元素添加到队尾
    • Dequeue 出队，将队首元素删除
    • 循环队列(Circular Queue)
    • 在Queue结构里面增加一个变量 size
    • 或者少用一格

    Tree

    定义

    • 树是N个节点和N-1条边的集合，0个或多个非空子树通过一条有向边连接，不能有环，否则就变成图了
    • 没有儿子的节点称为叶节点(leaf)
    • 有相同父亲的节点称为兄弟(sibllings)
    • 节点儿子数量称为这个节点的度(degree)
    • 从节点\(n_1\)到\(n_k\)的路径是唯一的，其长度是路径上边的数量
    • 对于节点\(n_i\)其深度为从根到它的路径长，而其高度等于它到叶节点的最大长度

    表示：

    • 数组存储：子节点数未知，不易表示。但完全二叉树可以，比如二叉堆。
    • FirstChild-NextSibling表示法
    • 记录一个子节点和它的兄弟节点
    • 因为儿子的顺序不固定，所以表示方式不唯一
    • 将FirstChild当作左儿子，NextSibling当作右儿子可以将任意一棵树表示成二叉树。 
      struct TreeNode
{
  ElementType Element;
  PtrToNode FirstChild;
  PtrtoNode NextSibling;
};
typedef struct TreeNode * PtrToNode

    Binary Tree

    • 每个节点最多有两个子节点的树
    • complete binary tree:
    • 除了最后一层都是满的
    • 最后一层节点都靠左
    • 第i层节点最多为\(2^{i-1}\)
    • 深度为k的二叉树最多有\(2^k-1\)个节点
    • \(n_0\)表示叶节点数，\(n_2\)表示度为2的节点数，则\(n_0 = n_2 +1\)

    Traversal

    • 先序遍历(preorder)
    • 根\(\rightarrow\)左\(\rightarrow\)右
    • 中序遍历(inorder)
    • 左\(\rightarrow\)根\(\rightarrow\)右
    • 后序遍历(postorder)
    • 左\(\rightarrow\)右\(\rightarrow\)根
    • 递归地进行，如果要改成迭代的话需要自己定义栈去模拟系统栈。

    线索二叉树

    教材上没有，但cyll书上有 目的是加速遍历 定义： * 如果一个节点的左儿子为空，那么它的左指针指向它的中序遍历的前驱节点 * 如果一个节点的右儿子为空，那么它的右指针指向它的中序遍历的后继节点 * 一定有一个head node,左儿子为根，右儿子为自身

    BST(Binary Search Tree)

    • 递归的定义：
    • 左子节点的值小于根节点
    • 右子节点的值大于根节点
    • 左右子树都是BST
    • 操作：
    • Find,查找指定值，递归地进行查询
    • Insert,插入一个新的值，先调用Find找合适位置再插入
    • Delete,删除指定值，先调用Find找到位置，再分两种情况操作，如果是子节点则直接删除，否则用左子树的最右侧节点或者右子树的最左侧节点覆盖。 时间度度均是\(O(h)\)如果考虑二叉树平衡则近似为\(O(logn)\)

    Heap(Priority Queue)

    • 用数组表示，但逻辑上是一颗完全二叉树
    • 操作：
    • Insert，插入一个数，向上调整到合适位置。
    • Delete，删除堆顶元素，然后把最后一个元素放到堆顶再向下调整。
    • BuildHeap,线性建堆，直接把一组数据读入数组，再从size/2处开始逐一向下调整。 时间复杂度：Insert 和 Delete 均为\(O(NlogN)\) 线性建堆时间复杂度当然是线性的

    Note

    其实我们在向上和向下调整时并不是进行交换操作，而是直接覆盖，本质上实在寻找合适的位置插入或者删除。原因是这样做更快，因为交换两个数据需要定义一个中间变量，要做三次赋值而覆盖只需要一次。

    Warning

    fds课程的考试中去年曾经出现过未明确建堆方式的编程题，不知道是该做若干次Insert还是直接线性建堆，这两种建堆方式的结果是不一样的，所以考试时要注意审题。zjk老师给的建议是如果题干没说清楚就选择线性建堆。

    Disjoint Set

    用于判断是否是等价类 用数组表示，如S[i]表示i的父节点序号，根节点用-size替换序号 操作： * Find 查找根节点序号 * Union 把两个集合并起来 1. 普通的Union,什么额外操作都不做 2. Union by size,把小的树并在大的树上 3. Union by height,把矮的树并在高的树上 4. Path Compression,每次合并都直接连到根上面 5. Union by size with Path Compression 6. Union by rank,因为路径压缩会改变高度，所以只需要知道高度近似值rank就行,相当于Union by height with Path Compression

    Graph

    一些基本概念：

    1. 无向图：边没有方向
    2. 有向图：边有方向
    3. 不考虑自环和多重边
    4. 联通：两个顶点之间有路径
    5. 强联通：针对有向图，两个顶点之间有双向路径
    6. 完全图：任意两个点之间都有边
    7. 对于无向图：
    8. 如果任意一对顶点都联通，则称这个无向图是联通的
    9. 联通分量：无向图的极大联通子图
    10. 边数最小的连通图是树
    11. 度数：分为入度和出度，入度是指向该顶点的边数，出度是该顶点指出的边数(针对有向图而言)；无向图的入度和出度肯定相等
    12. 图的表示：
    13. 邻接矩阵：缺点是空间开销大，在边少的情况下比较浪费内存，优点是查找快，写起来也更容易。如果是无向图的话邻接矩阵将会是对称的，所以可以将二维矩阵退化成一维的数组来节省空间。
    14. 邻接表：优点是避免了使用过长的连续内存，但缺点是查找起来相对较慢，并且你自己写的时候容易写出段错误。

    1.拓扑排序：

    Warning

    拓扑序不唯一，并且拓扑排序是排逻辑先后顺序。

    基本思路： - 寻找删除入度为零的顶点，更新其相邻顶点的入度 - 重复上述过程，直到图被删空

    Note

    需要一个容器去存放已删除节点，我们可以选择stack或者是queue。通常选择queue。

    2.不带权的最短路(BFS)

    基本思路： - 从给定起点开始，依次访问与该顶点相连的其他所有还未被访问的顶点，再以已访问的顶点为起点，重复上述操作直至所有顶点都被访问。

    3.带权最短路径算法 Dijkstra Algorithm

    基本思路： - 在所有未访问的顶点中，寻找一个当前和给定起点距离最短的顶点V(可以直接顺序查找也可以用一个min-heap) - 如果找到了，就将V标记为已访问；如果没找到，即所有顶点都已访问，则终止

    时间复杂度：顺序查找： \(O（|V|^2+|E|）\) 使用堆优化： \(O(ElogV)\)

    4.有负边的最短路 SPFA算法

    这个算法时间复杂度相当大\(O(VE)\) 

    void WeightNegative(Table T)

    5.AOE网络 (Active on Edge)

    • 规划活动进行的顺序
    • 有环无向图
    • 每个节点存在一个最早完成时间(earliest completion time)EC[v]和最晚完成时间(latest completion time)LC[v]
    • 每一条边存在一个持续时间(lasting time)也就是边权C，还有一个松弛时间(slack time)
    • 计算
    • \(EC[w]=max_{<v,w>}(EC[v]+C_{v,w})\)
    • \(LC[v]=max_{<v,w>}(LC[w]-C_{v,w})\)
    • 边的松弛时间为\(LC[]w-EC[v]-C_{v,w}\)
    • 关键路径是全为 0 松弛时间的边构成的路径

    6.网络流 Network Flow

    基本思路： - 在图G中找到一条从s到t的路径，可以使用BFS - 将这条路径的最短边长从路径中依次减去，更新为反向的边，将这个最短边的数值加入最大流量结果中 - 重复上述操作直到找不到从s到t的路径

    时间复杂度O（\(|E|^2log|V|\)）

    7.最小生成树 MST

    Prim算法(针对顶点)

    Warning

    最小生成树并不一定唯一，可以参考你的homework，如果存在权重相同的边就有可能会存在多个最小生成树。

    基本思路： - 任取一个起点 - 把当前距离起点最近且还未被访问的顶点纳入最小生成树，重复这个过程。 - 如果最后所有顶点都被纳入则找到一个最小生成树，反之则图不联通，不存在最小生成树。

    其实这个算法和Dijkstra算法很像

    Kruskal算法(针对边)

    基本思路： - 找当前图中还未被纳入生成树的最小边，如果这条边加入以后不会形成闭环则纳入这条边，反之则舍弃这条边。 - 重复这个过程直到顶点数-1条边被纳入

    时间复杂度O(\(|E|log|E|\))

    Note

    Kruskal算法刚需并查集去判断能否纳入找到的最小边，而Prim算法可以选择用数组去记录顶点是否已被纳入。 其次Kruskal算法需要对边进行排序，可以选择min-heap。

    8. DFS

    递归地进行，这个程序不要求掌握迭代版本的写法，实际上要写成迭代还挺麻烦的。 基本思路：直接上伪代码吧，感觉不太好表述 

    void DFS(Vertex V)
{
  visited[V] = true;
  for(each W adjacent to V)
  {
    if(!visited[V])
      DFS(W);
  }
}
    应用：寻找无向图中的联通分量 
    void ListComponents(Graph G)
{
  for(each V in G)
    if(!visited[V])
    {
      DFS(V);
      printf("\n");
    }
}

    9. Biconnectivity

    如果去掉一个点，剩下的图不再是联通的，则称这个点为 articulation point 求双向联通分量有两个常见算法，课上讲的应该是第一个

    Solution 1: Tarjan Algorithm

    基本思路： - 需要维护两个数组，一个是dfn，dfn(x)表是该顶点被dfs访问到的次序，即第几次调用dfs函数时被访问，也被称作时间戳；另一个是low，low(x)表示以x为起点能访问到的顶点中的最小时间戳，也被称作追溯值。 - 需要一个栈来存储已访问节点。 - 对于low的更新： * 初始时 low[x] = dfn[x] * 对于从 x 到 y 的边，如果在搜索树上,即y是x的子节点，则low[x]=min(low[x],low[y]); * 如果不在搜索树上，即y已经被访问过存在栈里了，则low[x]=min(low[x],dfn[y]) - 找关节点 * 如果u是root，则u是关节点当且仅当u至少有两个儿子 * 如果u不是root，则u是关节点当且仅当存在一个儿子v，满足dfn[u]<=low[v]

    Solution 2: Kosaraju Algorithm

    基本思路： - 创建反图，如果使用邻接矩阵存会比较容易。 - 两遍 dfs。 * 第一次 dfs，选取任意顶点作为起点，遍历所有未访问过的顶点，并在回溯之前给顶点编号，也就是后序遍历。 * 第二次 dfs，对于反向后的图，以标号最大的顶点作为起点开始 DFS。这样遍历到的顶点集合就是一个强连通分量。对于所有未访问过的结点，选取标号最大的，重复上述过程。
