    Log in and Authentication

    Cookie

    Info

    Cookie 是浏览器的一种本地储存方式，每条Cookie 的大小一般被限制为4kb。

    属性：

    Cookie在请求中以键值对的形式展现，形式为<name>=<value>

    生命周期：

    浏览器为cookie提供了过期设置，浏览器会分两类来管理token的周期： * Permanent cookie: + ExpiredAT:绝对时间 + MaxAge:设置时间+offset + 如果两者都有则以MaxAge为准 * Session cookie: 如果没有设置上述两个生命周期属性，浏览器会将其视作session cookie，其会在整个"session"内保留，直到"session"结束后删除。

    访问限制：

    浏览器提供了HttpOnly属性来限制前端主动访问cookie，这样cookie就只能在http请求中携带。但是浏览器知识限制了部分js接口，不代表这个属性绝对安全。 Secure 属性可以限制只有使用https协议的请求可以携带/设置cookie，这样可以尽可能防止cookie在传输过程中被泄露。 所以https的s其实是secure

    作用范围：

    Domain 和 Path 可以限制cookie所属的网站： * Domain 限制了cookie所属的域名。 * Path限制了cookie所属的子路径。 * 不同范围内可以有重名的cookie。

    CSRF和SameSite

    CSRF：跨站请求伪造 Site：一般来说，域名服务商只售卖xxx.xx这样的二级域名，比如你现在所看到的这个网页(https://kagunanana.site)，再扩展的多级域名都属于二级域名持有者。因此我们将二级域名相同的网址成为SameSite。 Origin:比site要严格很多，要求两个网址的schema,host,port完全相同。

    CORS

    CORS可以限制一个请求的来源和可访问资源。CORS在origin级别限制请求，其本质是基于请求HEADER的一种约定限制。

    密码学知识科普

    • Hash,利用难以求反函数的Hash函数来加密数据，不可逆
    • 对称加密，使用一个保密的秘钥，可以使用秘钥对信息进行解密
    • 非对称加密，常用算法有RSA ed25519(就是 git 那一章中生成ssh秘钥的算法)。有两个成对的秘钥，分为秘钥和公钥，其中一个秘钥加密的信息，只有用另一个秘钥才能解密。
    • 加盐。当Hash空间太小时，其实可以用暴力枚举的方式，生成一张反函数表来破解加密。所以，我们Hash时，一般都要在原信息中拼接一段随机信息来保证长度。这一串拼接的byte就被称作"salt"，这个过程也就叫做加盐。
    • 签名：一种防止公开消息被篡改的方式。常用的有MAC，可以理解为带密码的Hash,只有持有秘钥的人才能Hash出相同值。
    • TLS:HTTP 通过TLS协议保证安全性。
    • 对称+非对称混合加密
